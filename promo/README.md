# Promotion Playbook

Drafts and notes for launching `claude-skill-paul-graham` on LinkedIn
and X. Treat as a starting point — adapt to your real voice before
posting.

## Files

- [`linkedin-launch.md`](./linkedin-launch.md) — LinkedIn launch post
- [`x-launch-thread.md`](./x-launch-thread.md) — 7-tweet X thread

## Timing

| Day | Time (ET) | Why |
|---|---|---|
| Tuesday | 8–10am | Highest weekday LinkedIn engagement |
| Wednesday | 9–10am | Peak founder Twitter |
| Avoid | Friday afternoon, weekends | Lowest builder-audience engagement |

## Hashtag discipline

- **LinkedIn:** 4–5 max. Use `#ClaudeCode #Startups #PaulGraham
  #YCombinator #BuildInPublic`.
- **X:** zero. Hashtags reduce reach on X.

## Engagement plan (first 24 hours)

1. Post LinkedIn and X within 30 minutes of each other so the audience
   that follows you on both sees coordinated activity.
2. Reply to every comment in the first 4 hours. Reach is highly
   sensitive to early engagement velocity.
3. If anyone asks "what's a Claude skill?", link them to the README
   install section, not a separate explainer thread.
4. If someone suggests another thinker (Naval, Bezos, Andy Grove, etc.),
   reply with: "noted — adding to the shortlist; want to help curate?"
   This converts replies into future-collaborator pipeline.

## Pre-post checklist

- [ ] Repo is public and the README renders correctly on github.com
- [ ] Install instructions tested locally in a fresh shell
- [ ] At least one example in `examples/` reads convincingly
- [ ] `ATTRIBUTION.md` is in place; takedown contact is correct
- [ ] No personal info, API keys, or unrelated files committed
- [ ] LinkedIn post pasted into LinkedIn preview (don't post yet) to
      check line breaks and link card
- [ ] X thread loaded into Typefully / X drafts to check character
      counts (≤280 per tweet)

## Follow-up content (week 2 onward)

- **One-essay deep dive thread.** Pick one PG essay (e.g., "Schlep
  Blindness"), turn it into a 5-tweet breakdown with concrete examples
  from current startups. Link back to the repo.
- **"Which essay matches your stage" carousel** on LinkedIn — match
  founder stages (idea → first users → first hire → first raise) to
  the right PG essay.
- **Star milestone post.** When the repo hits ⭐100, post a brief
  thank-you with the next thinker you're packaging.

## Risk notes

- If Paul Graham (@paulg) responds critically, **do not argue**.
  Acknowledge, offer to remove anything contested, and adjust within
  24 hours. The repo's defensive position is fair-use commentary +
  links; it does not depend on his approval.
- Do not claim the repo "uses" or "trains on" PG's essays. It quotes
  short excerpts and links. Be precise about this in any reply.
